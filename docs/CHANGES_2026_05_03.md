# Changes Made to PRD - May 3, 2026

## Major Structural Changes

### 1. **Reversed Development Order**
- **Old**: v1 = OCR, v2 = GTO solver
- **New**: v1.0 = Interactive GTO solver (single scenario), v1.1 = Full GTO database, v2 = OCR
- **Why**: Validate solver pipeline works before tackling OCR complexity

### 2. **Removed Fixed Player Profiles** 
- **Old**: 4 fixed profiles (NIT, LAG, FISH, MANIAC) with pre-defined VPIP/PFR ranges
- **New**: 4 tightness levels (tight, gto, loose, extra_loose) based on villain's actual stats
- **Why**: Without nodelocking in postflop-solver, we can only adjust input ranges. Stat-based selection is more accurate and granular than fixed buckets.

### 3. **Changed Range Organization (Option B)**
- **Old**: Position first → `ranges/GTO/BTN/RFI_tight.txt`
- **New**: Tightness first → `ranges/tight/BTN/RFI.txt`
- **Why**: Cleaner file names, easier to work on all variants of same tightness together

### 4. **Range Source: GTO Wizard**
- **Source**: Download ranges from GTO Wizard (6-max, 100bb, all positions)
- **Format conversion required**: 
  - Input: `2d2c: 0.3094,2h2c: 0.3094,...` (specific combos, comma-separated)
  - Output: `22:0.3094` (generalized hands, line-separated) for postflop-solver
- **Process**: 
  1. Download 72 ranges from GTO Wizard → `ranges/raw/`
  2. Convert to postflop-solver format → `ranges/gto/`
  3. Generate tightness variants (tight, loose, extra_loose) from GTO baseline
- **Cost**: ~$50-100/month for GTO Wizard (can cancel after downloading)

## Stat-Based Exploitative System (v4)

### **Replaced Fixed Profiles With Stat-Based Range Selection**

**Stats Tracked** (priority order):
1. **Position-specific RFI%** - Directly determines opening range width
   - BTN_RFI, CO_RFI, MP_RFI, UTG_RFI, SB_RFI
   - Example: villain_BTN_RFI = 58% → use `loose/BTN/RFI.txt` (54-62% range)

2. **Position-specific defense%** - Directly determines defense range width
   - BB_vs_BTN, BB_vs_CO, BB_vs_MP, BB_vs_UTG, BB_vs_SB
   - Example: villain_BB_defense = 32% → use `tight/BB/defend_vs_RFI_BTN.txt`

3. **Position-specific 3bet%** - Determines 3betting range width
   - BB_3bet_vs_BTN, BTN_3bet_vs_CO, etc.

**Why positional stats matter**: A player might be tight from UTG (12% RFI) but loose from BTN (58% RFI). Fixed profiles can't capture this.

### **Exploitative Warnings**

Added warnings for clear tendencies:

**Calling Station Detection**:
- Condition: WTSD% >35% AND Fold to Cbet <40%
- Warning: "⚠️ Calling station (WTSD 38%) - Don't bluff"
- Action: Value bet more, bluff less

**Over-Aggressive Detection**:
- Condition: AF >4 AND Cbet >75%
- Warning: "⚠️ Over-aggressive (AF 5.2, Cbet 82%) - Bluff catch more"
- Action: Call down lighter, bluff catch with medium strength hands

### **Updated Overlay UI**

**New layout**:
```
┌──────────────────────────────────────────────────────────┐
│ Player: fish123                                          │
│ ⚠️ Calling station (WTSD 38%) - Don't bluff              │
│ Using: Loose defense range (52% vs 42% GTO)              │
│ Last action: Raised to $8.40 (84% pot)                   │
│ Hero hand: Ace-high flush                                │
│                                                           │
│ [Fold: 12%] [Call: 58%] [Raise 50%: 22%] [Raise 100%: 8%]│
└──────────────────────────────────────────────────────────┘
```

**Key changes**:
- Shows which tightness level being used (tight/gto/loose/extra_loose)
- Shows why it was selected (e.g., "High RFI at BTN: 58%")
- Shows exploitative warnings with actionable advice
- Shows "GTO vs GTO" if <100 hands sample size

## Range Library Updates

### **Naming Changes**

1. **BB defenses**: `vs_RFI_` → `defend_vs_RFI_`
   - Makes it clear these are BB defending ranges (with pot odds)
   - vs cold calls which have no pot odds

2. **Calling 3bets**: `vs_3bet_by_` → `call_vs_3bet_by_`
   - Clarifies these are CALL ranges after getting 3bet
   - Distinguished from 3betting ranges

3. **Calling 4bets**: `vs_4bet_by_` → `call_vs_4bet_by_`
   - Clarifies these are CALL ranges in 4bet pots
   - Distinguished from 4betting ranges

### **Removed Ranges**

**Removed cold calls** (3bet or fold is better strategy):
- MP: `cold_call_vs_UTG.txt` - removed (1 range)
- CO: `cold_call_vs_UTG.txt`, `cold_call_vs_MP.txt` - removed (2 ranges)
- SB: `cold_call_vs_UTG/MP/CO/BTN.txt` - removed (4 ranges)
  - SB is OOP postflop, should 3bet or fold

**Kept cold calls**:
- BTN: `cold_call_vs_UTG/MP/CO.txt` - kept (3 ranges)
  - BTN has best position, can profitably cold call

### **Rarity Markers**

Added markers to identify infrequent scenarios:

- **[RARE]**: <5% of hands
  - Most 4-bet call scenarios
  - Calls vs SB 3bets
  - SB 4betting

- **[VERY RARE]**: <1% of hands
  - MP/CO 4betting each other
  - SB 4betting vs blinds battles

**Purpose**: Prioritize building common ranges first, add rare scenarios later if needed.

### **Final Count**

**Per tightness level**: 72 ranges
- UTG: 11
- MP: 10 (removed cold_call)
- CO: 11 (removed cold_calls)
- BTN: 14 (kept cold calls)
- SB: 11 (removed cold calls)
- BB: 15 (defends)

**Total unique ranges**: 72 × 4 tightness levels = **288 ranges**

**v1.1 focus**: ~35 most common scenarios (no RARE markers) × 4 = **140 ranges**

## Key Insights

### **Why Stat-Based > Fixed Profiles**

1. **More accurate**: Uses actual observed behavior, not arbitrary buckets
2. **Granular**: 4 tightness levels per stat vs 4 total profiles
3. **Positional**: Captures that players are tight from UTG, loose from BTN
4. **Direct mapping**: villain_BTN_RFI = 58% → use 54-62% range (no guesswork)
5. **Same solve count**: ~12k-15k solves either way

### **Why Option B Organization**

1. **Cleaner paths**: `ranges/loose/BTN/RFI.txt` vs `ranges/GTO/BTN/RFI_loose.txt`
2. **Easier iteration**: Work on all "tight" ranges together, all "loose" together
3. **Runtime simplicity**: `level = select_level(stats)`, then `path = f"ranges/{level}/BTN/RFI.txt"`

### **Why Cold Calls Only from BTN**

1. **MP/CO**: OOP or marginal position → 3bet (leverage) or fold is better
2. **SB**: Worst position postflop → 3bet (leverage + fold equity) or fold
3. **BTN**: Best position → can profitably flat call to realize equity
